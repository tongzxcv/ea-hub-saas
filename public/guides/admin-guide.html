🔐 EA Hub - คู่มือผู้ดูแลระบบ (Admin)

เฉพาะผู้ดูแลระบบและโมเดเรเตอร์เท่านั้น

📋 บทนำ

Admin Panel ช่วยให้ผู้ดูแลระบบ (Admin) สามารถจัดการไฟล์, ผู้ใช้ และรายงานการละเมิดอย่างมีประสิทธิภาพ

หน้าที่หลักของ Admin:

  • ✅ ตรวจสอบและลบไฟล์ที่ไม่เหมาะสม
  • 👥 จัดการบัญชีผู้ใช้ (เปลี่ยน role, ดูประวัติ)
  • 🚩 ตรวจสอบรายงานการละเมิด (spamm, malware, ฯลฯ)
  • 📊 ติดตามสถิติและการใช้งาน

🚀 เข้าสู่ Admin Panel

  1. เข้าสู่ระบบด้วยบัญชี Admin
  2. ไปที่ https://godzalgohub.vercel.app/admin (หรือกดเมนู User → Admin Panel)
  3. จะเห็นหน้า Dashboard พร้อมแท็บ 3 อัน:
    • Files: จัดการไฟล์
    • Users: จัดการผู้ใช้
    • Reports: จัดการรายงาน
⚠️ ข้อมูลการ์ด: Admin Panel ปกป้อง RLS (Row Level Security) - คุณสามารถดำเนินการกับผู้ใช้อื่นได้เท่านั้น

📁 แท็บ Files - จัดการไฟล์

ดูและค้นหาไฟล์

  • ตารางแสดง: Title, Uploader, Category, Status, Downloads, Rating, Actions
  • ค้นหาด้วยชื่อไฟล์หรือผู้อัปโหลด
  • เรียงลำดับตามคอลัมน์ (กดหัวตาราง)

เปลี่ยนสถานะไฟล์

  1. กดคอลัมน์ "Status" ในแถวไฟล์ที่ต้องการ
  2. เลือกสถานะใหม่:
    • verified: ไฟล์ถูกอนุมัติและปลอดภัย
    • testing: ยังอยู่ระหว่างทดสอบ (ค่าเริ่มต้น)
    • stable: เสถียร พร้อมใช้งาน
    • experimental: ทดลอง อาจมีปัญหา
    • deprecated: ล้าสมัย ไม่แนะนำ
    • archived: เก็บถาวร (ซ่อนจากการค้นหา)

ลบไฟล์

  1. กลับไปที่แถวไฟล์ → กดปุ่ม 🗑️ Delete (มุมขวา)
  2. ยืนยันการลบ (บันทึกประวัติการลบ)
  3. ไฟล์และเวอร์ชันทั้งหมด ตลอดจนคอมเมนต์และรีวิว จะถูกลบ
💡 เมื่อใดควรลบไฟล์: Malware ที่ตรวจพบ, ลิงก์ที่ถูกเก็บถาวร, ไฟล์ scam/fake, ข้อมูลส่วนตัว, ความละเมิด DMCA

👥 แท็บ Users - จัดการผู้ใช้

ดูข้อมูลผู้ใช้

  • ตารางแสดง: Username, Email, Role, Created, Uploads, Actions
  • ค้นหา/กรอง: ชื่อผู้ใช้ หรือ email

เปลี่ยนบทบาท (Role)

  1. กดคอลัมน์ "Role" ในแถวผู้ใช้
  2. เลือกบทบาทใหม่:
    • member: ผู้ใช้ทั่วไป (ค่าเริ่มต้น) - อัปโหลดและรีวิว
    • uploader: อัปโหลดโดยไม่จำกัด (ในอนาคต)
    • moderator: โมเดเรเตอร์ - จัดการรายงาน (ในอนาคต)
    • admin: ผู้ดูแลระบบ - เข้าถึง Admin Panel

ดูการอัปโหลดของผู้ใช้

กดชื่อผู้ใช้หรือปุ่ม "View Uploads" เพื่อดูไฟล์ที่ผู้ใช้นั้นอัปโหลด

🚩 แท็บ Reports - จัดการรายงาน

ตรวจสอบรายงาน

  • ตารางแสดง: Reason, Type, Reported, Created, Status, Actions
  • ประเภท: File, Comment, Review
  • สาเหตุทั่วไป: Spam, Malware, Inappropriate, Copyright, Other

ดำเนินการต่อรายงาน

  1. คลิกแถวรายงาน เพื่อดูรายละเอียด
  2. ตรวจสอบ:
    • เรื่องที่รายงาน (ไฟล์/คอมเมนต์/รีวิว)
    • ความเห็นของผู้รายงาน
  3. ตั้งค่าสถานะเป็น:
    • resolved: จัดการแล้ว (ลบหรือแก้ไข)
    • dismissed: ไม่มีปัญหา ปล่อยไป
💡 ขั้นตอนที่แนะนำ: Spam/Malware → ลบไฟล์ + set status=resolved | Inappropriate → ลบคอมเมนต์ + resolved | False Report → dismissed

📊 สถิติและการตรวจสอบ

ข้อมูลที่แสดงใน Admin Panel (อัปเดตแบบ real-time):

  • 📁 Files: จำนวนไฟล์ทั้งหมด, status distribution
  • 👥 Users: จำนวนผู้ใช้ทั้งหมด, บทบาท
  • 📥 Downloads: ยอดรวมการดาวน์โหลด
  • ⭐ Ratings: ค่าเฉลี่ยดาว
  • 🚩 Pending Reports: รายงานที่ยังไม่ได้จัดการ

🔒 นโยบายและมาตรฐาน

ไฟล์ที่ต้องลบ

  • ⚠️ Malware, virus, trojan
  • ⚠️ Scam/Fake (ทำสัญญาเท็จ, ปลอมชื่อผู้พัฒนา)
  • ⚠️ ลิงก์ห้ามแท้ (ไฟล์ลบ/หมดอายุ)
  • ⚠️ ข้อมูลส่วนตัว (ชื่อ, ที่อยู่, หมายเลขโทรศัพท์)
  • ⚠️ ความละเมิด DMCA (ลิขสิทธิ์ผู้อื่น)
  • ⚠️ ไฟล์ที่มีพฤติกรรมสงสัย (ติดต่อผู้พัฒนา ให้อธิบายก่อน)

ไม่ควรลบ

  • ✅ ไฟล์ที่มีคะแนนต่ำ (ปล่อยให้ผู้ใช้ตัดสินใจ)
  • ✅ ความเห็นที่ไม่ชอบใจ แต่ไม่เป็นสแปม (ปล่อยให้เป็นอิสระ)
  • ✅ EA ที่ "อาจ" จำนวนแต่ยังไม่ได้พยายามลบหรือให้ Malware scan

⚙️ การดูแลประจำวัน

Daily Tasks

  • ✓ ตรวจสอบ Reports ที่ pending
  • ✓ ลบรายงานที่ resolved เก่า (เก็บถาวร)
  • ✓ ดูไฟล์ใหม่ที่อัปโหลด (เช่น "testing" status)
  • ✓ ตรวจสอบคอมเมนต์/รีวิว spam

Weekly Tasks

  • ✓ สรุปข้อมูล (Users, Files, Downloads)
  • ✓ ตรวจสอบผู้ใช้ที่ระงับหรือ ban (ถ้ามี)
  • ✓ ส่วนประกาศข้อมูล/แนวทางให้ชุมชน

❓ คำถามที่พบบ่อย

Q: ฉันลบไฟล์โดยไม่ได้ตั้งใจ สามารถเรียกคืนได้ไหม?

A: ในปัจจุบันไม่มีฟีเจอร์กู้คืน แต่ติดต่อผู้อัปโหลด ให้อัปโหลดใหม่ได้

Q: ควรลบคอมเมนต์ที่ไม่ชอบใจไหม?

A: ไม่ เว้นแต่มันเป็น spam/hate speech/malicious เท่านั้น - ให้อิสระในการแสดงความเห็น

Q: ผู้ใช้พยายามอัปโหลด Malware สองครั้ง ควรทำอะไร?

A: ลบไฟล์ + ติดตามผู้ใช้ + ส่งข้อความเตือน (ในอนาคต: ระงับ/ban account)

Q: รายงานใดๆเกี่ยวกับไฟล์เก่า

A: Set status=dismissed (ไม่มีเหตุผลที่จะจัดการ) อย่างไรก็ตาม ดูรายละเอียด ถ้า legitimate ให้แจ้งผู้อัปโหลด
